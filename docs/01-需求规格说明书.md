## 1. 项目概述

### 1.1 项目名称
嘉怡珠宝商城微信小程序（以下简称"嘉怡珠宝"）

### 1.2 项目背景
随着移动互联网和社交电商的快速发展，珠宝饰品行业正加速向线上转型。微信小程序凭借其"即用即走"、社交分享便利、支付闭环完整等优势，成为珠宝商家触达消费者的重要渠道。本项目旨在打造一个集商品展示、在线交易、会员运营、AI智能穿戴于一体的高端珠宝商城小程序。

### 1.3 目标用户群体
- **核心用户**：25-45岁女性，注重生活品质，有珠宝消费能力
- **次要用户**：珠宝爱好者、送礼需求用户（男性）
- **用户特征**：追求美观、注重品牌调性、喜欢社交分享

### 1.4 设计风格定位
- **整体风格**：轻奢、简约、精致、高级感
- **配色方向**：玫瑰金/香槟金为主色调，搭配白色/浅灰背景，深色文字
- **字体建议**：标题使用衬线字体（如思源宋体）体现优雅，正文使用无衬线字体（如苹方）保证可读性
- **设计关键词**：质感、留白、柔和光影、微动效

## 2. 项目目标

### 2.1 业务目标
- 建立品牌线上官方商城，实现24小时在线销售
- 通过会员积分体系提升用户复购率
- 借助AI穿戴技术降低用户决策成本，提升转化率
- 通过限时拍卖/活动模块制造紧迫感，促进成交

### 2.2 技术目标
- 小程序前端基于 uni-app 框架开发，支持后续扩展H5/App
- 后端基于 Java Spring Boot，提供稳定可靠的API服务
- 数据库使用 MySQL，确保数据安全与一致性
- 部署在腾讯云服务器，确保稳定可用

### 2.3 运营目标
- 后台可完全管理前台所有展示内容
- 提供完整的统计看板，辅助运营决策
- 支持灵活的营销活动配置

| 角色 | 描述 | 主要权限 |

## 7. AI穿戴模块详细需求

### 7.1 功能概述
AI穿戴模块允许会员用户上传自己照片，系统自动将商品（珠宝）叠加到用户照片上，生成穿戴效果图，帮助用户直观感受佩戴效果。

### 7.2 用户端流程

```
1. 用户进入商品详情页
2. 点击「AI穿戴」按钮
3. 判断是否会员 → 非会员提示"开通会员体验AI穿戴"
4. 判断当日剩余次数 → 用尽提示"今日已用完，明天再来"
5. 进入AI穿戴页
6. 选择拍照或从相册上传照片
7. 等待生成（loading动画，约5-15秒）
8. 展示生成结果（用户+珠宝合成图）
9. 可操作：保存图片 / 分享给好友 / 发布到发现页 / 直接购买
```

### 7.3 后端处理流程

```
用户上传照片 → 后端接收 → 检查会员资格+限额
→ 调取对应商品分类的提示词模板 → 替换变量（商品名/描述等）
→ 调用火山引擎豆包图生图API（doubao-seedream-5.0-lite）
  参数：用户照片 + 商品图 + 提示词
→ 接收生成结果 → 保存记录到数据库
→ 返回结果给用户 → 扣除当日限额
```

### 7.4 提示词模板策略

| 分类 | 提示词模板示例 |
|------|--------------|
| 戒指 | "一位优雅女性佩戴{product_name}戒指，{product_desc}，真实人物面部，自然光线，高清写实，珠宝细节清晰可见，真实感强" |
| 项链 | "一位女性佩戴{product_name}项链，{product_desc}，锁骨附近展示，自然光线，高清写实，珠宝质感细腻" |
| 耳饰 | "女性佩戴{product_name}耳饰，面部侧面展示，{product_desc}，自然光线，皮肤真实质感，珠宝闪耀" |
| 手链/手镯 | "女性手腕佩戴{product_name}手链/手镯，{product_desc}，细节展示，自然光线，高清写实" |

### 7.5 AI API选型方案

| 项目 | 说明 |
|------|------|
| 推荐平台 | 火山引擎（豆包大模型） |

| 参考成本 | 日均100次≈22元/天，月均≈660元 |
