## 1. 方案概述

### 1.1 业务描述
AI穿戴模块允许会员用户上传个人照片，系统通过AI图像生成技术，将用户照片与珠宝商品图进行融合，生成用户佩戴该珠宝的效果图。该功能旨在降低用户购买决策成本，提升购物体验和转化率。

### 1.2 核心流程

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 用户上传  │ → │ 检查会员  │ → │ 调用AIAPI │ → │ 返回结果  │
│  照片     │    │  + 限额   │    │  生成     │    │  展示     │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
```

**选择理由**：
1. **多图融合能力强**：支持用户照片+商品图同时输入，融合效果自然
2. **性价比高**：0.22元/张，在同类产品中价格最低
3. **速度快**：5-15秒出图，用户体验好
4. **兼容OpenAI格式**：接入成本低，无需额外学习
5. **字节生态**：与抖音视频链接功能同属字节系，后期可能有更多联动

## 4. 接口对接

### 4.1 火山引擎豆包API接入

#### 4.1.1 注册与获取Key
1. 访问 [火山引擎控制台](https://console.volcengine.com)
2. 注册企业账号并完成实名认证
3. 在"火山方舟"中创建API Key
4. 新用户通常有免费试用额度

#### 4.1.2 API调用示例

```java
// 使用 RestTemplate 调用豆包图生图API
public String generateAiWear(String userPhoto, String productImage, String prompt) {
    String url = "https://ark.cn-beijing.volces.com/api/v3/images/generations";
    
    Map<String, Object> requestBody = new HashMap<>();
    requestBody.put("model", "doubao-seedream-5-0-lite");
    requestBody.put("prompt", prompt);
    requestBody.put("image", userPhoto);     // 用户照片
    requestBody.put("image2", productImage);  // 商品图
    requestBody.put("size", "2K");
    requestBody.put("response_format", "url");
    requestBody.put("watermark", false);      // 无水印
    
    HttpHeaders headers = new HttpHeaders();
    headers.setContentType(MediaType.APPLICATION_JSON);
    headers.setBearerAuth("YOUR_API_KEY");
    
    HttpEntity<Map<String, Object>> entity = new HttpEntity<>(requestBody, headers);
    ResponseEntity<Map> response = restTemplate.postForEntity(url, entity, Map.class);
    
    // 解析返回的图片URL
    List<Map<String, Object>> data = (List<Map<String, Object>>) response.getBody().get("data");
    return (String) data.get(0).get("url");
}
```

#### 4.1.3 图片处理要求

| 项目 | 要求 |
|------|------|
| 用户照片格式 | JPEG/PNG |
| 用户照片大小 | ≤10MB |
| 用户照片分辨率 | 建议720×1280以上 |
| 商品图格式 | PNG（透明背景最佳） |
| 商品图大小 | ≤5MB |
| 多图数量 | 最多2张（用户照+商品图） |

### 4.2 图片上传方案

#### 4.2.1 前端上传
- 用户从微信相册选择或拍照
- 使用 `wx.chooseImage` 获取图片
- 使用 `wx.compressImage` 压缩图片（控制在2MB以内）
- 使用 `wx.uploadFile` 上传到后端

#### 4.2.2 后端存储
- 使用腾讯云COS对象存储
- 临时目录：`/ai-wear/temp/{userId}/{timestamp}.jpg`
- 结果目录：`/ai-wear/result/{userId}/{recordId}.jpg`
- URL有效期：临时文件24小时，结果文件长期保存

## 5. 提示词工程

### 5.1 提示词策略

提示词是AI生成质量的关键。根据不同商品分类设计差异化提示词：

#### 戒指类
```
一位优雅女性佩戴{product_name}戒指，{product_desc}，手指特写，
自然光线，高清写实，珠宝细节清晰可见，真实皮肤质感，柔和肤色，
戒指在手指上自然贴合，真实感强，不夸张
```

#### 项链类
```
一位优雅女性佩戴{product_name}项链，{product_desc}，
颈部与锁骨区域展示，自然光线，高清写实，肤色自然，
项链贴合颈部线条，珠宝质感细腻闪耀，真实佩戴效果
```

#### 耳饰类
```
女性佩戴{product_name}耳饰，{product_desc}，
面部侧面近景展示，自然光线，高清写实，皮肤真实质感，
耳饰在耳部自然垂坠，闪耀质感，自然美观
```

#### 手链/手镯类
```
女性手腕佩戴{product_name}，{product_desc}，
腕部细节展示，自然光线，高清写实，珠宝贴合手腕，
皮肤真实质感，自然美观，细节清晰
```

### 5.2 变量说明

| 变量 | 来源 | 示例值 |
|------|------|--------|
| {product_name} | 商品名称 | "璀璨星辰钻戒" |
| {product_desc} | 商品描述（截取前50字） | "18K金镶嵌天然钻石" |
| {style} | 商品风格标签 | "优雅/简约/奢华" |

### 5.3 提示词优化建议

1. **正向词**：包含"真实感、自然光线、高清、写实"等关键词
2. **负向词**：排除"卡通、夸张、变形、模糊、水印"等（通过API参数控制）
3. **动态调整**：根据用户性别调整（女性→男性）
4. **A/B测试**：同一分类可配置多个提示词模板，对比生成效果

## 6. 成本估算

### 6.1 按调用量估算

| 日均调用量 | 月均调用量 | 月费用 | 适合阶段 |
|-----------|-----------|--------|---------|
| 50次/天 | 1500次/月 | ~330元 | 初期测试 |
| 100次/天 | 3000次/月 | ~660元 | 正常运营 |
| 200次/天 | 6000次/月 | ~1320元 | 活跃期 |
| 500次/天 | 15000次/月 | ~3300元 | 高峰期 |

### 6.2 成本控制策略

| 策略 | 说明 |
|------|------|
| 每日总限额 | 后台可设置全平台每日最大生成次数 |
| 会员分级限制 | 不同等级每日限额不同（普通3次/黄金5次/钻石10次） |
| 结果缓存 | 同一用户+同一商品不重复生成（7天内） |
| 低峰期预热 | 热门商品可在低峰期提前生成效果图 |
| 免费额度利用 | 火山引擎新用户通常有数百万Tokens免费额度 |

## 7. 限流与优化

### 7.1 限流方案

```
┌─────────┐    ┌──────────┐    ┌──────────┐
│ 用户提交 │ → │ 检查会员  │ → │ 检查每日  │
│ AI生成   │    │ 等级权限  │    │ 剩余次数  │
└─────────┘    └──────────┘    └──────────┘
                              ↓ 通过
                    ┌─────────────────────┐
                    │ 检查全平台今日总限额  │ → 超过 → 返回"今日已满"
                    └─────────┬───────────┘
                              ↓ 通过
                    ┌─────────────────────┐
                    │ 提交生成任务 + 扣除  │
                    │ 次数(先扣后生成)     │
                    └─────────────────────┘
```

### 7.2 MySQL表设计（补充）

```sql
-- 每日限额重置（定时任务）
-- 每天凌晨00:00:00执行
UPDATE user SET ai_wear_count = 0, last_ai_reset_time = NOW();

-- 查询用户剩余次数
SELECT 
    ml.ai_wear_daily_limit - u.ai_wear_count AS remaining
FROM user u
LEFT JOIN member_level ml ON u.level_id = ml.id
WHERE u.id = ?;
```

### 7.3 异步处理方案

为避免用户长时间等待，支持同步/异步两种模式：

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| 同步模式 | 用户等待生成完成（5-15秒），实时返回 | 默认模式，体验好 |
| 异步模式 | 提交后返回任务ID，稍后查询结果 | AI服务不稳定时降级 |

### 7.4 用户体验优化

1. **Loading动画**：生成过程中展示品牌定制加载动画，缓解等待焦虑
2. **进度提示**：显示"正在识别照片→正在融合珠宝→即将完成"等步骤
3. **失败重试**：生成失败时自动重试（最多2次），不消耗用户次数
4. **结果预览**：左右滑动对比原图与AI结果
5. **引导购买**：生成完成后显示商品购买入口
